Abstract

AGRe (Academic Gossip-based Recommender) is a website that generates recommendations for learning resources (books, on-line documentations, software tools, existing projects, public code repositories, and others), important persons (e.g., colleagues, tutors) to be contacted, good/bad points regarding the personal knowledge on the taught topics, optimal hours of study, etc.

Introduction

As our society becomes ever more complex people will require more and more information. To accommodate the need for expertise in ever more diverse and specific areas online resources for learning keep multiplying. Universities are also providing open access to their courses through the internet and online only courses keep increasing in number. But the amount of resources available also makes it difficult to find the ones best suited to you and your needs.

Identifying the right materials and people to start learning can have a significant impact on what a student learns and how fast he does it. AGRe will try to make smart recommendations to the user about which books they should read, what courses they should take, who are the experts in the field and what other resources are available on the web.

To do this AGRe combines several data sources about books, universities, scientific publications and other web content. The content from several places is aggregated and filtered to better fit the user's query and information we have about them. Information such as the users location could be used to better identify which universities are nearby so courses and professors from these universities would be ranked higher in the search results. Information about the user's language preferences could be used to identify content that is easier for the user to understand etc.

To generate good recommendations exploiting all the information we have about the user and the data is important. To do this we must create user profiles, track information about what queries are made and what resources are used. We can combine this information to generate the show the most appropriate content for that specific user using a technique called Collaborative Filtering. This technique takes into account the users previously used resources and recommends learning materials that users who have used similar resources have selected but this user has not.

To allow this project to be reused and understood by machines the results will integrate information about the type of resource in each result and various properties related to it. To do this we use RDF metadata incorporated into the HTML in format called RDFa. We will also be using popular open ontologies such as schema.org to provide an easily understood structure to the data that can be combined with and incorporated into other applications that use the same ontologies.

To make interacting with our system as easy as possible we will create a bot that tries to accommodate talking using natural language. The bot will look for keywords and use natural language attributes such as dependencies to identify the topics the user is querying and then query AGRe and display the results in a way that fits the context in which it is being used. The bot will remember what the user said in the past and incorporate that into the resulting query. The bot will also aid in creating the user profiles by asking the user questions and saving the results.

For machines to be able to use AGRe we need to provide a different interface that relies less on visual elements or natural language but is still expressive enough to allow the use of all of AGRe's functionalities. For this we will provide a SPARQL endpoint. The SPARQL language allows developers to create queries that are specific to their needs by providing the query along with optional parameters that filter the data. Another advantage of SPARQL is its ability to query related objects along with specific properties.

Our high-level goals are to:

  • To access a comprehensive list of resources for learning using other APIs and public datasets.
  • Combine and filter the resulting list for specific queries.
  • Generate user and resource profiles and use them to make useful recommendations.
  • Embed semantic data so as to be part of the Semantic Web.
  • Provide easy to use interfaces for both humans and machines.

We hope AGRe will be useful in both helping learners by providing the right materials and highlighting the best content and people so that useful resources will continue to be developed and used. This objective is important now and is becoming increasingly more so as new technologies are developed and companies create new roles that demand more specialization. So we hope that AGRe will be easy to integrate with future applications so that as many people as possible get to become their best selves.

Resources

Courses

Courses are units of teaching that typically lasts one academic term, are led by one or more instructors and are usually taught at universities.

  • Model: Course and CourseInstance (from schema.org)
  • Useful APIs and other sources:
    • mooc-list
    • Open Education Consortium website
  • Motivation: students as most people are familiar with them, universities are commonly ranked and can be used to determine their importance, professors are generally guaranteed to be qualified and accreditation of its completion can be provided
  • Downsides: prohibitive costs, the need to be present at a specific location and the limitation that they, almost always, tackle larger subjects, span several months with a fixed schedule (solution: Massive open online courses (MOOCs) alleviate most of these problems although their degrees tend to hold less value)

Books

Books are written or printed work consisting of pages glued or sewn together along one side and bound in covers. eBooks are digital scans of books usually in the form of a PDF file.

  • Model: Book (from schema.org)
  • Useful APIs and other sources: Google Books API
  • Motivation:
    • books are one of the most successful ways we have stored knowledge throughout history
    • the wealth of knowledge stored in books is something everyone can make use of when trying to learn something
    • books can cover subjects in detail
  • Downsides: can suffer from being outdated, poorly written or too expensive (solution: there are many eBooks available for free which can be accessed conveniently at any point during the learning process)

Scholarly Articles

Scholarly or peer-reviewed articles are written by experts in academic or professional fields.

  • Model: ScholarlyArticle (from schema.org)
  • Useful APIs and other sources: Elsevier APIs
  • Motivation:
    • excellent sources for finding out what has been studied or researched on a topic as well
    • find bibliographies that point to other relevant sources of information
    • sometimes the only resources available about a subject are in the form of scholarly articles
    • tend be in-depth analyses on a very specific topics
  • Downsides: can suffer from being outdated (solution: lots of new information is published this way but there are also reviews of past work that show the current state of research on a topic)

People

Professors and other experts are people with a lot of knowledge about certain topics and, in the case of professors, engage in teaching others about it and conducting research on their chosen topics.

  • Model: Person (from schema.org)
  • Useful APIs and other sources:
    • we can identify them as authors of books or scholarly articles
    • we can identify them through the courses they teach
    • DBpedia
    • identify information about them from and about the institutions they are associated with through the Google Places API
  • Motivation:
    • finding the right person to teach you can have a huge impact when learning
    • the right person can show you how interesting a topic can be and make learning about easier and more fun

Other Web Content

There is a lot of useful information on the Web that falls outside of the aforementioned categories.

  • Model: WebSite and WebPage (from schema.org)
  • Useful APIs and other sources:
    • GitHub API
    • Stack Exchange API
  • Motivation:
    • Github is a site that maintains repositories for various projects, most of which are software related. There are many repos which contain example projects and list useful resources which are very useful for learning at all levels of expertise
    • Stack Exchange is a network of question-and-answer (Q&A) websites on topics in diverse fields, each site covering a specific topic, where questions, answers, and users are subject to a reputation award process. The reputation system allows the sites to be self-moderating
    • simple topics can be covered in the scope of an answer to a question on one of the Stack Exchange sites
    • some questions simply provide a starting point on were to start learning about a topic

Arhitecture

The main server will be written in Python using the Django framework. For SPARQL and RDFa funcionality we will use RDFlib. For querying the APIs we will use the requests library and for scraping recevied data when neccesary we will use BeautifulSoup. For parsing natural language we will use spaCy and Wordnet using NLTK's interface. We will also use Surprise for its implementation of Collaborative Filtering. For the frontend we will use React to make an UI bot and website along with a SPARQL endpoint.

Motivation of using these technologies and platforms

Python

  • is an interpreted high-level dynamically typed programming language for general-purpose programming
  • tries to emulate natural languge thus easing the use
  • tries to remain terse which makes it useful for fast develoment

Django

  • high-level Python Web framework that encourages rapid development and clean, pragmatic design
  • provides quite a lot of functionality which is also scalable
  • comes integrated with its own templating language
  • most significant feature is the speed with which applications can be build with it

Pyhon Libraries

  • Requests is an elegant and simple HTTP library for Python that makes it easy to querry data from the web
  • Beautiful Soup is a Python package for parsing HTML and XML documents. It creates a parse tree for parsed pages that can be used to extract data from HTML, which is useful for web scraping
  • RDFLib is a Python library for working with RDF, a simple yet powerful language for representing information. This will allow us to leverage the power of the semantic web to make our application as reusable as possible.

NLP libraries

  • spaCy can be used to process natural language queries from a UI bot and extract useful information from plain text. It was built with speed and accuracy in mind choosing to implement and offer good implementations of state-of-the-art techniques. The features that we'll find most useful is its dependency parser and interface to word2vec embeddings
  • Wordnet lexical database is another useful resource in natural language processing which consits of synsets that have associated various words and relationships. We can use Wordnet to try to extend queries and make word level inferences

Recommender system

Surprise is a Python scikit building and analyzing recommender systems. It provides various ready-to-use prediction algorithms for generating reccomendations. Among those algorithms are various implementations for collaborative filtering. Collaborative filtering is a method of making automatic predictions (filtering) about the interests of a user by collecting preferences or taste information from many users (collaborating). The underlying assumption of the collaborative filtering approach is that if a person A has the same opinion as a person B on an issue, A is more likely to have B's opinion on a different issue than that of a randomly chosen person. We will use people's uses of various resources as indication of their preferences and try to find resources that other users who have used similar resources have used.

React

  • is a JavaScript library for building user interfaces
  • is widely used by many developers and has become one of the most popular javascript framework ever
  • allows a developer to create reusable components through javascript
  • we choose this framework because its versatility and because of the many resources available for it

UI bot

  • is a computer program which conducts a conversation via auditory or textual methods
  • can be used to offer a way to interact with machines that is closer to the way people interact with each other

SPARQL

  • is an RDF query language—that is, a semantic query language for databases—able to retrieve and manipulate data stored in Resource Description Framework (RDF) format
  • allows us to provide a powerful way to query our application that makes use of all the structure provided by our use of RDFs and filter specific information which gets rid of unneeded transfers of data and consolidates multiple queries in a traditional REST API

GitHub API

General Flow and User Interface

AGRe is a platform designed for both students and professors. Each of them has different priveleges.

If a new user and wants to register in AGRe, he will be needed to fill-in some questions from which our ML module will extract information regarding the Interests (and Tags) a user has. Based on those Interests on the Query Frame or Home page there will be recommended, used by teacher's rank some resources, such as Books, Articles, Online Courses etc.

The flow diagram presents the main actions and processes along a user's session.

Query Frame/Home Page

  • has a menu bar at the top where the user can edit its account or talk to UI bot (see Fig. 3)
  • at the left can be seen a panel with a Search functionality from where the user could search new resources, these resourcces are collected from our SPARQL endpoint which communicates with our third party APIs and the resulted resources are processed after that by our ML module
  • each resource presented in the Resource Panel can be accesed, reviewed and saved in the pocket of the current user
  • on the right panel there are presented relevant persons that can be contacted if the user would consider it can use his help
  • on the bottom right it's presented a panel which provides insights (like course duration, timetable, reiews of other students) regarding the course identified through the user's search

UI bot interface

  • the UI bot would provide a chatting functionality which will help the user find new resources
  • the UI bot will use NLP in order to process the query of the user and will generate a search query that will be given to the Search module and post-processed by the ML module
